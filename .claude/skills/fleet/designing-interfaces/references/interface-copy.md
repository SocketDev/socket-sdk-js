# Interface Copy

Use this for UI labels, buttons, errors, empty states, loading text, confirmations, and navigation names.

## Read the Situation

- State the user’s context, action, constraint, and next step.
- Match tone to the user’s mental state: calm for errors, brief for success, specific for recovery.
- Prefer plain language over jargon and internal terms.

## Write the Copy

- Buttons: verb + object. Avoid OK, Submit, Yes, and No.
- Errors: say what happened, why it matters, and how to fix it.
- Empty states: explain why the area is empty and what to do next.
- Loading text: explain what is happening and how long it may take when that helps.
- Confirmations: name the exact action and the consequence.
- Navigation: use names users understand, not product internals.

## Tighten the Wording

- Cut filler and repeated context.
- Use active voice.
- Keep one sentence to one idea.
- Make destructive, permission, and recovery states explicit.

## Check the Result

- Can a new user act without guessing?
- Can a busy user scan the text in a second?
- Does every message point to the next action?
- Does the copy stay consistent across the flow?

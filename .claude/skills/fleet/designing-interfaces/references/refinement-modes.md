# Refinement Modes

Use these when the design direction is already locked and the work is about tuning the
experience, not re-choosing the product.

## Choose the smallest useful dial

- `clarify` — make the action, outcome, or state obvious. Tighten labels, CTAs, empty states,
  errors, and any ambiguous copy.
- `distill` — remove anything that does not support the primary task. Fewer words, fewer
  controls, fewer competing surfaces.
- `harden` — cover loading, empty, error, permission, read-only, retry, and i18n states.
  Keep the happy path intact while closing the gaps.
- `onboard` — help a new user reach first value. Reduce activation friction, add guidance where
  needed, and keep the path short.

## Use these to tune tone

- `bolder` — increase contrast, scale, or structural confidence when the interface feels timid.
- `quieter` — remove noise, decoration, and over-assertion when trust or readability matters.
- `delight` — add one memorable detail only after the core flow works and the detail serves the
  product.
- `animate` — use motion to explain state, feedback, or spatial change. Respect reduced motion.

## Use these to tune composition

- `typeset` — improve hierarchy, line length, font pairing, and reading rhythm.
- `layout` — fix spacing, alignment, density, and visual flow before adding new decoration.
- `colorize` — give a monochrome interface a deliberate color system instead of random accents.
- `adapt` — verify the result across breakpoints, density changes, and locale expansion.

## Remember

- Tune one axis at a time.
- Prefer a narrow fix over a broader redesign.
- Validate the result in the browser before calling it done.
